# 飞书接口配置说明

## 概述

本项目使用飞书开放平台 API 进行多维表格读写、文档创建和消息推送。本文档说明如何配置飞书自建应用及相关权限。

## 一、飞书自建应用创建步骤

### 1. 创建应用

1. 登录飞书开放平台：https://open.feishu.cn/
2. 进入「开发者后台」→「企业自建应用」→「创建应用」
3. 填写应用信息：
   - 应用名称：无人配送反馈系统
   - 应用描述：用于收集和处理无人车用户反馈
   - 上传应用图标（可选）

### 2. 配置应用权限

在「权限管理」→「权限列表」中搜索并添加以下权限：

| 权限名称 | 权限标识 | 说明 |
|----------|----------|------|
| 读取多维表格 | bitable:table:readonly | 读取工单池数据 |
| 写入多维表格 | bitable:table:write | 写入工单数据 |
| 创建文档 | docx:document:create | 创建周报文档 |
| 发送群消息 | im:message:send | 推送周报通知 |

### 3. 获取应用凭证

在「凭证与基础信息」中获取：

- **App ID**：应用唯一标识
- **App Secret**：应用密钥（保密！）

### 4. 配置 IP 白名单（可选）

如果部署在固定服务器，可配置 IP 白名单提高安全性。

## 二、多维表格配置

### 1. 创建工单池表格

1. 飞书 → 云文档 → 新建 → 多维表格
2. 命名：「无人配送反馈工单池」

### 2. 获取表格 Token

1. 打开多维表格
2. 点击右上角「分享」→「高级设置」
3. 复制「链接」，格式如：
   `https://www.feishu.cn/drive/folder/fldxxxxxxxxxxxxxxxxx`
4. `fld` 后面的部分即为 **BITABLE_APP_TOKEN**

### 3. 获取 Table ID

1. 在多维表格中打开「开发者模式」（右上角三个点 → 开发者模式）
2. 在浏览器控制台执行：`console.log(window.currentTableId)`
3. 输出值即为 **BITABLE_TABLE_ID**

### 4. 配置表格字段

按 AGENTS.md 中的 Schema 创建字段：

| 字段名 | 类型 | AI 字段 | 说明 |
|--------|------|---------|------|
| feedback_id | 文本 | 否 | 工单唯一标识 |
| channel | 单选 | 否 | 反馈渠道（7个枚举值） |
| user_tier | 单选 | 否 | 用户分层（6个枚举值） |
| category | 单选 | 是 | 分类（安全/故障/体验/投诉/建议） |
| priority | 单选 | 是 | 优先级（P0/P1/P2/P3） |
| status | 单选 | 否 | 状态（待处理/处理中/待回访/已关闭） |
| vehicle_id | 文本 | 否 | 车辆 ID |
| city | 文本 | 否 | 城市 |
| content_raw | 多行文本 | 否 | 反馈原文 |
| content_summary | 多行文本 | 是 | AI 摘要 |
| created_at | 日期 | 否 | 创建时间 |
| closed_at | 日期 | 否 | 关闭时间 |
| assigned_to | 人员 | 否 | 处理人 |
| csat_score | 数字 | 否 | 满意度评分 |
| contact_name | 文本 | 否 | 联系人姓名 |
| contact_phone | 文本 | 否 | 联系人电话 |
| contact_allowed | 单选 | 否 | 是否允许回访 |
| location_detail | 文本 | 否 | 位置详情 |

## 三、环境变量配置

在项目根目录创建 `.env` 文件，填写以下配置：

```ini
# DeepSeek API（用于聚类分析）
DEEPSEEK_API_KEY=your_deepseek_api_key

# 飞书自建应用凭证
FEISHU_APP_ID=cli_xxxxxxxxxxxxxxxx
FEISHU_APP_SECRET=xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx

# 飞书多维表格配置
BITABLE_APP_TOKEN=fldxxxxxxxxxxxxxxxxx
BITABLE_TABLE_ID=tblxxxxxxxxxxxxxxxxx

# 飞书群配置（可选，用于周报推送）
FEISHU_CHAT_ID=oc_xxxxxxxxxxxxxxxx

# SerpAPI（用于舆情搜索，可选）
SERPAPI_KEY=your_serpapi_key
```

## 四、权限配置注意事项

1. **安全第一**：`.env` 文件已被 `.gitignore` 忽略，**永远不要提交真实密钥**
2. **最小权限原则**：只授予应用必要的权限
3. **权限审批**：部分权限需要管理员审批后才能生效
4. **测试环境**：建议在测试租户中先配置和测试，再部署到生产环境

## 五、常用 API 端点

| 功能 | API 端点 | 方法 |
|------|----------|------|
| 获取租户 Token | `/open-apis/auth/v3/tenant_access_token/internal` | POST |
| 读取表格记录 | `/open-apis/bitable/v1/apps/{app_token}/tables/{table_id}/records` | GET |
| 创建表格记录 | `/open-apis/bitable/v1/apps/{app_token}/tables/{table_id}/records` | POST |
| 创建文档 | `/open-apis/docx/v1/documents` | POST |
| 发送群消息 | `/open-apis/im/v1/messages?receive_id_type=chat_id` | POST |

## 六、故障排查

### 常见错误码

| 错误码 | 原因 | 解决方案 |
|--------|------|----------|
| 99991663 | 应用未授权 | 检查权限是否已添加并审批通过 |
| 99991665 | Token 过期 | 重新获取 tenant_access_token |
| 100114 | 表格不存在 | 检查 BITABLE_APP_TOKEN 和 BITABLE_TABLE_ID |
| 100121 | 字段不存在 | 检查字段名称是否正确 |

### 调试建议

1. 使用 `scripts/check_feishu_auth.py` 检查鉴权是否正常
2. 使用 `scripts/list_bitable_fields.py` 检查表格字段是否正确
3. 检查飞书开放平台的「事件订阅」和「接口调用日志」
