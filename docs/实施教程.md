# 全模块实施教程（小白版）

按模块顺序做，每个模块开头标注了负责人和依赖关系。遇到报错：复制报错原文问自己的 AI 助手（Trae/Cursor），它已读过 AGENTS.md，知道项目上下文。

## 模块 0：全员准备（半天内完成）

1. 飞书：报名通过后按赛事邮件激活专属租户，三人互加进一个群。
2. GitHub：两位队友把用户名发给队长，队长在仓库 Settings → Collaborators → Add people 邀请；队友邮箱接受后执行：
   ```
   git clone https://github.com/AGD-h/feishu-neolix-AIfeedback.git
   ```
3. Python 环境（1号、3号需要）：先在终端试 `py --version`，能显示版本号就已装好；否则去 python.org 下载安装（**务必勾选 "Add python.exe to PATH"**）。然后在项目目录执行：
   ```
   py -m pip install -r requirements.txt
   ```
   注意：Windows 上如果 `python` 命令没反应（微软商店空壳别名），一律改用 `py`。
4. DeepSeek（1号先办）：注册 platform.deepseek.com → 充值 20 元 → 创建 API Key → 复制 `.env.example` 为 `.env`，把 Key 填进 `DEEPSEEK_API_KEY=`。**`.env` 不要发群里，更不要提交 git。**

## 模块 1：工单池多维表格（2号 · MVP 核心 · 最先做）

系统的心脏。所有其他模块都依赖它，2号第一周全力做这个。

### 1.1 建表

1. 飞书 → 云文档 → 新建 → 多维表格，命名「无人配送反馈工单池」。
2. 删掉默认字段，按 AGENTS.md 第四节的 Schema 逐个建字段：
   - `feedback_id`：文本
   - `channel`：单选，选项照抄 Schema 的 7 个枚举值
   - `user_tier`：单选，6 个枚举值
   - `category`：单选，5 个值
   - `priority`：单选，P0/P1/P2/P3（给 P0 设红色、P1 橙色，仪表盘好看）
   - `status`：单选，4 个值，默认「待处理」
   - `vehicle_id` / `city`：文本
   - `content_raw` / `content_summary`：多行文本
   - `created_at` / `closed_at`：日期（含时间）
   - `assigned_to`：人员
   - `csat_score`：数字

### 1.2 配 AI 字段（分类、分级、摘要）

1. 把 `category` 字段改为「AI 生成」类型（字段编辑 → 智能字段/AI 字段，入口名称随版本略有差异）。
2. 提示词模板（`priority`、`content_summary` 同理改写）：
   > 你是无人配送车公司的客服质检员。根据反馈原文 {content_raw}，判断它属于哪一类：安全（涉及事故/碰撞/危险驾驶/监管）、故障（车辆无法工作/趴窝/柜门打不开）、体验（响应慢/取件麻烦/误报）、投诉（明确表达不满要求处理）、建议（提出改进想法）。只输出类别名。
3. priority 的提示词要把 AGENTS.md 里 P0–P3 的定义原样写进去。
4. 用 3 条手造数据测试：填一条「车撞到人了」应出 P0/安全；「取件码收不到」应出 P2/体验。不准就改提示词。

### 1.3 录入表单

1. 多维表格顶部 → 表单视图 → 新建表单，勾选字段：channel、user_tier、city、vehicle_id、content_raw。
2. 开启「允许非租户内用户填写」，复制公开链接——这就是电话/微信群反馈的人工转录入口，也是一车一码的基础。

### 验收

在表单里提交一条反馈 → 表格里出现新行 → AI 字段自动填出分类/分级/摘要。

## 模块 2：仿真数据集（1号 · MVP · 与模块 1 并行）

仓库里已有 `data/gen_mock_data.py`（第一版已写好），直接用：

1. 项目目录下运行：
   ```
   cd data
   py gen_mock_data.py
   ```
2. 产出 `data/output/mock_feedback.csv`（约 800 条）和 `data/output/分布说明.md`。
3. 打开 CSV 抽查 20 条：内容是否像真人说的话、P0 是否极少、时间是否集中在早晚高峰。
4. 想调整数量/分布/话术：直接改脚本顶部的配置区，或让 AI 助手改。
5. 交付给 2号：把 CSV 发到团队群，2号在多维表格 → 导入 → Excel/CSV → 字段一一对应导入。

### 验收

多维表格里出现几百条带完整字段的数据，仪表盘（模块 3）有东西可算。

## 模块 3：自动化流转 + 回访 + 仪表盘（2号 · MVP 第二周）

### 3.1 自动分派

多维表格 → 自动化 → 新建流程，建 4 条规则：

| 触发条件 | 动作 |
|---------|------|
| 新记录且 priority=P0 | 给「区域负责人」（演示=队员A）发飞书加急消息 + 群卡片 |
| 新记录且 priority=P1 | 给「运维值班」（演示=队员B）发消息，assigned_to 自动填 B |
| 新记录且 priority=P2/P3 | assigned_to 填「客服」（演示=队员C） |
| status 变为「待回访」 | 给填报人/处理人发满意度表单链接 |

### 3.2 SLA 超时升级

自动化支持定时触发：每 30 分钟扫描一次，条件「status=待处理 且 created_at 超过 SLA 时限」→ 给上级（另一位队员）发升级提醒。做不到精细就演示 P1 一档即可，路演讲清思路。

### 3.3 满意度回访

再建一个只含 feedback_id 和 csat_score（评分题 1–5）的表单；处理人把 status 改为「待回访」时自动化发出表单；回收后人工或自动回填 csat_score，status 改「已闭环」。

### 3.4 仪表盘

多维表格 → 仪表盘 → 新建，放 6 个图表：

1. 指标卡：总工单数、已闭环数、闭环率
2. 柱状图：按 channel 分布
3. 饼图：按 category 分布
4. 折线图：按日新增工单
5. 表格：当前未处理的 P0/P1 清单
6. 指标卡：平均 csat_score

### 验收（= MVP 整体验收）

现场演示：表单提交一条 P0 反馈 → AI 自动分类分级 → 队员A 手机收到加急 → A 处理并改 status → 回访表单发出 → 评分回填 → 仪表盘数字变化。全程无人工改字段。

## 模块 4：一车一码（1号 · 增强阶段第一件事 · 零代码版）

1. 拿到模块 1.3 的表单公开链接。
2. 飞书表单支持 URL 参数预填：链接后拼 `?prefill_vehicle_id=NX-BJ-0233&prefill_channel=scan_qr`（参数名以实际表单字段为准，测试一次）。
3. 每台"演示车"生成一个专属链接 → 用草料二维码等工具转成二维码 → 贴在车模/展板上。
4. 演示话术：路人扫码 → 手机上打开反馈页（车辆 ID 已自动带上）→ 提交 → 工单池实时出现。
5. 若要更漂亮的自定义页面，再做 `h5/` 的自研版（Vercel 免费托管 + 飞书 API 写入），非必需。

## 模块 5：舆情采集（1号 · 增强阶段）

思路：公开网页内容 → DeepSeek 清洗成 Schema 格式 → CSV → 导入工单池。

1. 让 AI 助手按 AGENTS.md 写 `data/search_public_opinion.py`，对它说：
   > 我是1号，按 AGENTS.md 写 search_public_opinion.py：输入是一个我手动整理的公开评论文本文件（每行一条），调 DeepSeek 把每条清洗成工单 Schema 的 CSV 行（channel=social_media，user_tier 由内容判断），输出到 data/output/。
2. **建议起步用「手动整理 + AI 清洗」**：自己从抖音/微博/黑猫投诉复制 50–100 条真实评论存成 txt，脚本只做清洗。这完全够演示，且零合规风险。自动抓取等有余力再说。
3. 跑完把 CSV 导入工单池，演示「社媒渠道」数据也进了同一个池子。

## 模块 6：聚类周报（3号 · 增强阶段）

1. 前置：2号 在飞书开放平台（open.feishu.cn）→ 创建自建应用 → 开通多维表格读取权限 → 把 App ID/Secret 给 3号 填进 `.env`，同时把工单池的 app_token 和 table_id（在表格 URL 里）填进 `.env`。
2. 让 AI 助手写 `report/weekly_report.py`，对它说：
   > 我是3号，按 AGENTS.md 写 weekly_report.py：读取多维表格近 7 天工单，用 DeepSeek 聚类出 TOP5 高频问题，每个问题给出现次数、典型原声、产品改进建议，输出 Markdown 到本地。
3. 先跑通「输出本地 Markdown」，再加「自动发飞书群」（机器人 webhook 最简单：群设置 → 群机器人 → 自定义机器人 → 复制 webhook 地址，POST 一段 JSON 即可）。
4. 演示话术：这就是「反馈回流产品改进」的闭环证据——运营周报自动告诉研发「本周雨天急刹投诉上升 40%」。
