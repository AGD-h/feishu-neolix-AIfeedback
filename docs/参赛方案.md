# 依托飞书 AI 的无人配送全渠道用户反馈闭环运营体系

**参赛队伍：** 新石器 AI 反馈先锋队  
**命题方：** 新石器无人车  
**技术平台：** 飞书 AI（多维表格 + AI 字段 + 自动化 + 智能体）  
**日期：** 2026 年 7 月

---

## 一、项目背景

### 1.1 行业痛点

无人配送车正加速走入城市社区、校园、园区，但随之而来的用户反馈管理面临三大挑战：

**渠道分散，难以统一管理**  
用户反馈散落在微信群、客服电话、抖音/微博评论、滴滴订单评价、车身二维码、车端告警等多个渠道，运营团队需要在不同平台之间来回切换，响应慢、遗漏多。

**人工分拣效率低，分级依赖经验**  
传统模式下，客服人员逐条阅读反馈、人工分类分级，不仅耗时，而且主观性强——同一条反馈，不同客服可能给出完全不同的优先级判断，导致 P0 安全事故被遗漏、P3 咨询被过度响应。

**反馈止于运营，无法回流产品**  
大量有价值的用户声音淹没在工单系统中，产品团队只能凭"感觉"判断哪些问题需要优先改进，缺乏数据驱动的决策依据。

### 1.2 新石器的诉求

新石器作为无人配送行业的领军企业，需要一套能够**快速落地、低成本验证**的反馈运营体系，实现：

- 全渠道反馈统一接入
- AI 自动分类分级分派
- 高频问题自动聚类
- 产品改进数据驱动

## 二、解决方案

### 2.1 核心理念：零代码为主，轻代码为辅

我们的方案以**飞书多维表格**为核心载体，零代码搭建工单池、AI 字段、自动化流程和仪表盘；仅在三个环节用轻量 Python 脚本做补充：仿真数据生成、社媒舆情清洗、聚类周报自动化。

**为什么选飞书？**

| 能力 | 飞书提供 | 我们的用法 |
|------|---------|-----------|
| 数据存储 | 多维表格 | 统一工单池，替代传统工单系统 |
| AI 处理 | AI 字段（分类/分级/摘要） | 替代人工客服分拣 |
| 流程自动化 | 自动化流程 | 自动分派、超时升级、满意度回访 |
| 数据可视化 | 仪表盘 | 运营看板，实时看数据 |
| 交互式分析 | AI 问数 | 产品团队自由深挖数据 |
| 协同沟通 | 飞书群 + 消息推送 | 预警通知、周报推送 |

## 三、核心功能模块

### 3.1 模块一：统一工单池（多维表格 + AI 字段）

**18 个标准字段的工单 Schema**，覆盖从"反馈进来"到"闭环结束"的全生命周期：

| 类别 | 字段 | 说明 |
|------|------|------|
| 标识 | feedback_id | 工单唯一编号 |
| 来源 | channel / user_tier | 7 种渠道 × 6 种用户分层 |
| 内容 | content_raw / content_summary | 原文 + AI 摘要 |
| AI 处理 | category / priority | 5 类分类 × P0-P3 四级优先级 |
| 流程 | status / assigned_to / created_at / closed_at | 状态流转追踪 |
| 车辆 | vehicle_id / city / location_detail | 车辆与位置信息 |
| 回访 | csat_score / contact_name / contact_phone / contact_allowed | 满意度与联系信息 |

**AI 字段提示词示例（分类）：**

> 你是无人配送车公司的客服质检员。根据反馈原文 {content_raw}，判断它属于哪一类：安全（涉及事故/碰撞/危险驾驶/监管）、故障（车辆无法工作/趴窝/柜门打不开）、体验（响应慢/取件麻烦/误报）、投诉（明确表达不满要求处理）、建议（提出改进想法）。只输出类别名。

### 3.2 模块二：自动化运营闭环

| 自动化规则 | 触发条件 | 动作 |
|-----------|---------|------|
| P0 安全加急 | 新记录且 priority=P0 | 给区域负责人发飞书加急消息 + 群卡片 |
| P1 运维分派 | 新记录且 priority=P1 | 通知运维值班，自动填 assigned_to |
| P2/P3 客服分派 | 新记录且 priority=P2 或 P3 | assigned_to 自动填"客服" |
| SLA 超时升级 | status=待处理 且 超过时限 | 给上级发升级提醒（30 分钟轮询） |
| 满意度回访 | status 变为"待回访" | 自动发送满意度评分表单 |

### 3.3 模块三：仪表盘

6 张图表构成运营看板：

1. **指标卡**：总工单数、已闭环数、闭环率
2. **柱状图**：按渠道分布
3. **饼图**：按问题类别分布
4. **折线图**：按日新增趋势
5. **表格**：当前未处理的 P0/P1 清单
6. **指标卡**：平均满意度评分

### 3.4 模块四：交叉聚合聚类周报

这是项目的**创新亮点**——不是用一个 AI 模型包打天下，而是把两种 AI 能力交叉使用：

| 能力 | 工具 | 特点 | 用途 |
|------|------|------|------|
| 基础统计 | 飞书多维表格仪表盘 | 零代码、实时更新 | 看分布、看趋势 |
| 自动聚类 | DeepSeek API | 定时运行、标准化输出 | TOP 高频问题识别、趋势信号、改进建议 |
| 交互深挖 | 飞书 AI 问数 | 人在飞书里自由提问 | 归因分析、关联分析、细分对比、异常检测 |

**周报内容结构：**

```
无人配送反馈周报
├── 本周概况（工单总数 + 多维表格链接）
├── 自动聚类洞察（DeepSeek）
│   ├── 本周高频问题 TOP 5（每个问题含：名称/类别/优先级/次数/典型原声/改进建议）
│   ├── 趋势信号（上升最快的问题类型、新出现的问题）
│   └── 产品改进建议汇总（按优先级排序）
├── 飞书 AI 问数 · 推荐问题
│   ├── 归因分析类
│   ├── 趋势预测类
│   ├── 关联分析类
│   ├── 细分对比类
│   └── 异常检测类
└── 交叉聚合总结（三者各司其职，引导用户用对工具）
```

## 四、创新点

### 4.1 交叉聚合 AI 架构

不是"一个大模型解决所有问题"，而是让不同的 AI 工具做各自最擅长的事：

- **飞书仪表盘**做统计（精确、实时、零代码）
- **DeepSeek**做跨记录聚类（自动化、标准化，适合定期生成周报）
- **飞书 AI 问数**做交互式深挖（灵活、上下文感知，适合产品经理自由探索）

三者通过周报串联：DeepSeek 告诉你"本周有哪些问题"，飞书 AI 问数帮你"深入挖掘每个问题的细节"。

### 4.2 零代码核心 + 轻代码补充

系统主体（工单池、AI 字段、自动化、仪表盘）完全在飞书里搭建，运营团队无需写代码就能维护和调整。

只有三个环节用 Python 脚本做补充：
- 仿真数据生成（800 条，用于演示和测试）
- 社媒舆情清洗（DeepSeek 把公开评论转成标准工单格式）
- 聚类周报自动化（定时拉取数据 → 聚类 → 推送）

### 4.3 数据契约驱动协作

全团队共用**一套工单 Schema**（18 个字段），无论是仿真数据、舆情清洗、还是周报读取，都严格遵守同一个数据契约。三名队员分工协作但数据标准统一，避免了"各做各的、最后拼不起来"的问题。

## 八、预期成果与价值
