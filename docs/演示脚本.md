## 场景 1：扫码反馈 → AI 自动分类分级（45 秒）

### 🎬 操作流程

1. **拿起手机扫描车身二维码**（或点击预备好的表单链接）
2. **填入反馈内容**：
   - channel：扫码反馈（已自动带入）
   - vehicle_id：NX-BJ-0233（已自动带入）
   - city：北京
   - content_raw：`小区里车速有点快，建议限速，孩子多的地方危险`
3. **点击提交**
4. **切到投影屏 → 刷新多维表格** → 顶部出现新记录
5. **指着新记录说明**：
   - `category` 字段自动填为「投诉」← **AI 字段自动完成**
   - `priority` 字段自动填为「P3」← **AI 字段自动完成**
   - `content_summary` 自动生成摘要 ← **AI 字段自动完成**

### 🎤 口播话术

> 「这是第一层闭环——**反馈接入 + AI 自动分拣**。
>
> 路人扫车身二维码，手机上打开反馈页，**车辆 ID 已经自动带上了**。填一句话："小区里车速有点快，建议限速"。
>
> 提交。切到工单池——新记录已经出现。**注意看这三个字段**：分类、优先级、摘要，全部是飞书 AI 字段自动填出来的，**没有任何人工操作**。
>
> AI 判断这是「投诉 / P3」——正确。这就是 70% 人工分拣工作量的替代。」

## 场景 2：P0 安全事故 → 加急触达（45 秒）

### 🎬 操作流程

1. **在多维表格中新增一条记录**（或通过表单提交）：
   - content_raw：`车过路口没让行人，差点撞到推婴儿车的，太危险了`
2. **等待 3-5 秒**（AI 字段处理 + 自动化触发）
3. **AI 字段自动填出**：category=安全，priority=P0
4. **自动化触发** → 区域负责人手机收到飞书加急消息
5. **举起手机展示加急消息** → 群内同时出现群卡片

>
> 提交一条："车过路口没让行人，差点撞到推婴儿车的"。
>
> AI 识别为 **安全 / P0**——红色高亮。**自动化流程立刻触发**：
>
> - 区域负责人手机收到飞书加急消息（举手机展示）

## 应急预案

### ⚠️ AI 字段识别延迟

**现象：** 提交反馈后 AI 字段几秒内没填出来
**应对：**
- 口播过渡：「AI 字段正在处理，我们顺便看下仪表盘的实时数据……」
- 刷新页面，通常 5-10 秒内会填出
- 如果超时，用预先准备好的「已填好」记录展示

### ⚠️ DeepSeek 调用失败

**现象：** 周报脚本运行时 DeepSeek 报错
**应对：**
- 脚本会自动兜底，输出"DeepSeek 调用失败"提示但其他流程正常
- 展示预先准备好的周报文档（已生成好的链接）
- 口播：「现场网络波动，我们看下昨天预生成好的周报……」

### ⚠️ 飞书群消息未收到

**现象：** 周报推送后群里没收到消息
**应对：**
- 检查 FEISHU_CHAT_ID 配置
- 手动在群里 @ 机器人触发
- 直接展示周报文档链接

### ⚠️ 网络中断

**应对：**
- 预先录制完整 Demo 视频（预录屏）
- 现场播放视频，口播同步
- 「为了让大家看得更清楚，我们准备了高清录屏……」

## 预录屏脚本（赛前录制）

### 录制要求

- 分辨率：1920×1080
- 格式：MP4
- 时长：5-7 分钟
- 录制工具：OBS / 飞书录屏 / Windows 自带 Game Bar

### 录制流程

1. **开场白**（10 秒）：项目名称 + 队伍
2. **场景 1**：扫码反馈 + AI 字段自动填（45 秒）
3. **场景 2**：P0 加急触发（30 秒）
4. **场景 3**：仪表盘 6 图表（30 秒）
5. **场景 4**：周报生成 + 群推送（90 秒）
6. **结尾**：架构图 + 价值总结（30 秒）

### 录制注意事项

- 鼠标移动要慢、稳，关键位置停留 2 秒
- 终端命令提前打字，避免现场输入错误
- 飞书文档滚动要平滑，配合口播节奏
- 字体调大（浏览器缩放 125%），投影清晰
- 录制完导出后，在投影仪上测试一遍清晰度
